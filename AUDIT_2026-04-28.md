## 2. Strengths confirmed during review

The hardening work captured in `SECURITY_AUDIT_CHECKLIST.md` is genuinely
present in the code:

- **Tenant isolation.** `createScopedClient` (`src/services/supabase.ts:41`)
  builds a per-request anon-key + Bearer-JWT client; migration 008 enables
  `FORCE ROW LEVEL SECURITY` on `influencers`, `outreach`, `consent`,
  `dsar_requests` with `auth.uid()` policies. The service-role client is
  reserved for documented admin paths (token blocklist writes, DSAR status
  updates, retention purge, erasure cascade).
- **JWT verification.** `authenticate` (`src/middleware/auth.ts`) requires a
  `jti` claim, pulls verification material from a pluggable
  `KeyProvider` (env / KMS / Secrets Manager / RS256 PEM / JWKS), and pins
  the algorithm list per provider — preventing algorithm-confusion attacks.
  Production refuses to boot the symmetric `env` provider unless
  `ALLOW_INSECURE_KEY_PROVIDER=true` (`src/validateEnv.ts:56`).
- **Token revocation.** `tokenBlocklist` is persistent (Postgres-backed) with
  fail-closed in-memory caches and explicit cache statistics
  (`src/services/tokenBlocklist.ts`).
- **PII at rest.** `FieldEncryptionService` implements AES-256-GCM with KMS
  envelope encryption and zeroes plaintext DEKs after use
  (`src/services/fieldEncryption.ts`). PII columns are tagged in
  migration 007.
- **Privacy controls.** Consent-gated processing
  (`src/middleware/requireConsent.ts`), DSAR endpoints, GDPR-Art-17 cascade
  in `eraseUserData` (`src/services/privacy.ts:288`), and audit-log payload
  redaction via an explicit non-PII allow-list
  (`src/middleware/auditLog.ts:redactAfterState`).
- **Transport / parser hardening.** `helmet` with locked-down CSP, HSTS
  (2 y + preload), `no-referrer`, `same-origin` COOP/CORP; strict
  `application/json`-only body parser (`src/middleware/contentType.ts`);
  CORS allow-list with rejection logging; trust-proxy configurable.
- **Rate limiting.** Global IP limiter (100 req / 15 min), dedicated
  `/health/ready` limiter (30 req / min), per-user `bulkSearchLimiter`
  (5 / 15 min) and `logoutLimiter` (10 / 15 min) keyed on JWT `sub`.
- **Outbound HTTPS only.** Axios request interceptor in
  `src/services/scrapeCreators.ts:49` blocks any non-HTTPS URL — meaningful
  SSRF-downgrade defence.
- **Input hygiene.** `sanitize-html` strips all tags, control characters and
  bidi-override codepoints are removed, NFKC normalisation, prototype-pollution
  keys filtered, recursion depth capped at 16 (`src/middleware/sanitize.ts`).
- **Logging.** `pino` with custom `safeErrSerializer` strips Axios / Supabase
  payloads; redacts `Authorization`, `x-api-key`, `Cookie`, `Set-Cookie`,
  `connectionString`, secret env names. Structured request IDs propagate to
  the error handler.
- **Container.** Multi-stage Dockerfile, base image pinned by SHA-256 digest,
  `npm ci --ignore-scripts` in both stages, dedicated non-root `appuser`.
- **CI.** Build + test on Node 20 & 22, `npm audit --audit-level=high`,
  `npm audit signatures`, CodeQL weekly, dependency-review on every PR,
  Dependabot for npm + actions + docker, readiness gate enforcing the
  checklist score.

## 3. Findings (independent observations)

No High or Medium issues uncovered. The items below are Low / informational
— mostly hardening polish and code-quality nits — and would not block a
production deployment on their own.

### 3.1 Low

- **L-A — `postcss` transitive moderate advisory.** Resolve with
  `npm audit fix` (devDependency only; runtime image is unaffected).
- **L-B — Loose date validation.** `validateOutreach`
  (`src/middleware/validate.ts:101-108`) uses `Date.parse()`, which accepts
  loosely-formatted strings such as `"December 17, 1995"`. For DB
  consistency, validate against a strict ISO-8601 regex (or `zod`'s
  `z.string().datetime()`).
- **L-C — Late env validation.** `index.ts` performs `import { createApp }`
  before `validateEnv()`, so missing required envs surface as the
  module-level guard in `services/supabase.ts` rather than the friendlier
  `validateEnv()` message. Move `validateEnv()` (and `assertSupabaseAdminSignOut`)
  to the very first statements after `dotenv/config`, or make the supabase
  client lazy.
- **L-D — No per-user rate limit on single-handle search.**
  `POST /api/influencers/search` is gated only by the global IP limiter.
  An authenticated abuser sharing 100 IPs could still drive paid-API quota
  through this endpoint. Mirror the `bulkSearchLimiter` pattern with a
  per-`sub` limiter (e.g. 30/min/user).
- **L-E — `auditLog` only intercepts `res.json`.**
  (`src/middleware/auditLog.ts:144`). Today every controller returns JSON,
  but a future `res.send`/`res.end`-based handler would silently bypass
  audit logging. Wrap `res.send` and `res.end` defensively, or add a
  CI guard against direct `res.send` use on protected routes.
- **L-F — CORS rejection surfaces as 500.** `corsOptions.origin` calls
  `callback(new Error(...))`, which the global error handler renders as a
  generic 500. The SIEM warning compensates, but a 403 with explicit `error`
  body would be more accurate and cheaper to triage.
- **L-G — `revoked_tokens` not part of the erasure cascade.** Tokens are
  keyed by JTI (no `user_id` column), expire naturally, and are not
  user-identifying on their own — so this is acceptable. Document the
  reasoning in `eraseUserData` to pre-empt future audits asking the same
  question.
- **L-H — Field encryption disabled is logged at INFO.**
  `getFieldEncryptionService()` (`src/services/fieldEncryption.ts:296`) emits
  an `INFO` line when `FIELD_ENCRYPTION_KMS_KEY_ID` is unset. Promote to
  `WARN` (or `ERROR`) when `NODE_ENV === 'production'` so a missing
  configuration is loud, not silent.
- **L-I — Empty-string `notes`/`niche` accepted.** `validateUpdate`
  (`src/middleware/validate.ts:73`) admits `""`. Decide whether empty means
  "clear" (replace with `null`) or "invalid" — current behaviour persists
  the empty string, which downstream UIs may render as a phantom badge.

### 3.2 Code quality / hygiene

- **Q-A — `@types/node@^25` vs runtime `node:20-alpine`.** Type definitions
  may permit APIs unavailable at runtime. Either align `@types/node` to
  `^20.x` or upgrade the base image to `node:22-alpine` (current active
  LTS). Dependabot will track the digest.
- **Q-B — `axios` could be replaced with built-in `fetch`.** Node 20+ ships
  `undici`. Removing `axios` shrinks the supply-chain surface (one fewer
  direct prod dep, plus its transitive `form-data`, `proxy-from-env`, etc.).
  Optional.
- **Q-C — `jest --forceExit --detectOpenHandles` papers over leaks.** The
  flags are pragmatic but mask the real cause (likely the `setInterval`
  cleanup timers in `tokenBlocklist` + KMS providers, both of which already
  call `unref`). Worth a short investigation so CI doesn't depend on
  `--forceExit`.
- **Q-D — No coverage threshold enforcement.** `jest.config.js` collects
  coverage but does not set `coverageThreshold`. Add a baseline
  (e.g. 80% lines / 70% branches) so CI catches regression in test
  coverage on top of the readiness gate.
- **Q-E — Migration 008 schema drift.** Migration 002 originally created
  `UNIQUE(handle, platform)`; migration 008 drops and re-adds as
  `(handle, platform, user_id)`. Functionally correct, but consolidating
  into a single canonical migration (or adding an explicit comment in 002
  pointing to 008) helps fresh deployments.
- **Q-F — `validateConsent`, `validateUpdate`, etc. duplicate enum lists.**
  The legal status / channel / consent type values live in `types/index.ts`
  and again as runtime arrays in `middleware/validate.ts`. Derive the
  runtime arrays from the type definitions (or vice versa) so a future
  type addition cannot drift from validation.
- **Q-G — `bulkSearchInfluencers` writes by index into a sparse array.**
  `results[idx] = ...` assignments inside async tasks rely on JS sparse
  semantics; subsequent `results.filter((r) => r && r.success)` works
  correctly, but reading code is easier with an explicit pre-allocation:
  `const results = new Array(handles.length);`.

### 3.3 Compliance & operations

- **C-A — Retention defaults vs Privacy Policy.** Defaults
  (`src/services/privacy.ts:381`) are 90 d audit log, 30 d revoked tokens,
  365 d DSAR. Verify these mirror the periods stated in
  `docs/PRIVACY_POLICY.md` / DPA so the published policy is enforceable.
- **C-B — Partial-erasure handling.** `eraseUserData` returns 207
  Multi-Status when one of the five steps errors but does not retry. Add
  retry-with-backoff or surface the partial state to an admin queue so the
  request can be completed within the GDPR 30-day SLA.
- **C-C — `purgeData` is fire-and-forget.** The endpoint runs the cascade
  inside the request handler. For a multi-table purge this is fine today
  but will eventually need to move to a scheduled / background job
  (e.g. Supabase pg_cron) to keep request latency predictable.

## 4. Threat-model spot checks

- **JWT alg confusion** — pinned per provider (`getAlgorithms()`); `none`
  cannot be selected. ✓
- **JWT replay after logout** — JTI required + persistent blocklist + global
  Supabase admin signOut on logout. ✓
- **Authorisation bypass via service role** — restricted to documented
  admin code paths; user-facing reads/writes go through `req.scopedClient`. ✓
- **Cross-tenant reads** — `influencers_select_own` etc. in migration 008
  guard at the DB layer in addition to controller-level filters. ✓
- **SSRF / outbound abuse** — Axios HTTPS interceptor + Zod URL schema
  enforcing `https://`; URLs returned by ScrapeCreators that fail validation
  are stripped (`.optional().catch(undefined)`). ✓
- **Stored XSS via scraped bio** — `sanitizeBody` runs after parsing; the
  scraper-returned values are also length/URL constrained. ✓
- **Prototype pollution** — explicit `__proto__`/`constructor`/`prototype`
  drop in `sanitizeObject`. ✓
- **Timing side-channels on admin endpoints** — `authorize('admin')` runs
  before payload validators on the DSAR PATCH route (M4 fix verified). ✓
- **CSP gap on error pages** — CSP is `default-src 'none'`; the API never
  serves HTML. ✓
- **Log injection / secret leakage** — pino redaction list covers both
  Axios and Supabase error shapes. ✓
